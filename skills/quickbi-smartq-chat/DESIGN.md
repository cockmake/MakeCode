# 小Q问数（quickbi-smartq-chat）— Design Document

## Background

Quick BI 平台提供了小Q问数开放 API，支持用户通过自然语言对数据集或上传的数据文件进行智能查询分析。此前用户需要手动调用 API、处理 SSE 流式响应、解析 OLAP 数据并渲染图表，流程复杂且门槛高。

本技能将这一完整流程封装为 Agent 可调用的脚本，使用户只需用自然语言提问，即可自动完成数据集匹配、流式问数、数据取数和图表渲染，大幅降低数据分析门槛。

**目标用户**：需要对 Quick BI 数据集或本地 Excel/CSV 文件进行数据分析的业务人员和开发者。

## Design Decisions

- **统一技能，双模式设计**：将数据集问数和文件问数合为一个技能，通过用户是否上传文件自动切换模式，避免用户需要区分两个不同的技能
- **自动智能选表**：数据集问数模式下无需用户提供 cubeId，脚本自动查询用户权限数据集并通过智能选表 API 匹配，降低使用门槛
- **试用凭证自动注册**：未配置凭证时自动基于设备唯一标识注册试用用户并回写配置，实现开箱即用
- **脚本内置异常检测与提示**：将试用到期、无权限、文件解析失败等异常的检测逻辑和用户提示文案内置到脚本中，确保 Agent 能原样转达标准化的引导信息
- **matplotlib 本地渲染**：使用 matplotlib 在本地渲染图表 PNG，避免依赖远端渲染服务，保证离线可用性

## Dependencies

- Quick BI 小Q问数开放 API（SSE 流式接口）
- Quick BI OLAP 查询 API
- Quick BI 组织用户管理 API
- Python 库：requests, pyyaml, matplotlib, numpy
